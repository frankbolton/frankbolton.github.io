Setting up an Ubuntu 16.04 server for remote access to do Deep learning on your local network

The back story is as follows.... On my desk I have a decent PC with a mediocre GPU, and I want to use a larger tower with a decent GPU in headless mode (no display) and stashed in a corner. I want to use a remote connection. First off- I need to attribute the sources that I combined before going through my setup.
Puget Systems: The Best Way To Install Ubuntu 16.04 with NVIDIA Drivers and CUDA
Changjiang's blog : Install Nvidia CUDA-9.0 on Ubuntu 16.04 for Deep Learning
Digital Ocean - How to Install and Configure VNC on Ubuntu 16.04
Digital Ocean - How To Install the Anaconda Python Distribution on Ubuntu 16.04
Coderwall- setting up remote connection to notebook

Step by Step instructions

  1. Set up the hdd partitions and install Ubuntu server as listed in Puget link above. Complete thru Step 4.
  2. Set up SSH on the server and connect from a second computer (unplug monitor when rebooting).
  3. Install CUDA and associated libraries as in Changjiang above, complete all the steps (note- there's a GOTCHA where one of the commands will fail and expects an explicit "Y"
  4. Setup the NVIDIA utities using the Puget blog post (From "Configure CUDA for use on your system" to the end.
  5. Validate that you have NVidia drivers installed by running 'nvidia-smi' from the terminal.This will confirm that your driver version is correct
  6. Validate that CUDA 9.0 is installed by running 'nvcc --version'
  7. Congrats - now it's time to install Python and other fun things. We'll go the Miniconda path
  8. Download the latest anaconda package and check the hash
  9. Run the installer and then use the coderwall instructions to get a remote connection to the notebook
  10. Open a notebook and confirm you can import the regular packages (numpy, pandas, matplotlib, scikit-learn)
  11. Install tensorflow as described here.
  12. GOTCHA - jupyter notebook isn't installed by default- so import tensorflow fails. run this in the tensorflow venv "conda install jupyter"
  13. Validate that Tensorflow can be imported to the ipython notebook (also check that you are using the right interpreter using '!which python'
  14. Clone the environment into a new Keras env
  15. Activate the new environment and then perform 'pip install keras'
  16. Clone your tensorflow environment a second time to create an environment called pytorch
  17. Activate the pytorch environment and install pytorch using the command generated using the "start locally" section of the pytorch page, (e.g. 'conda install pytorch torchvision -c pytorch').
